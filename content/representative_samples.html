I’m also on point to list out a representative sample of Content API queries I’d anticipate.   I’ll use a sql-ish syntax since it’s compact and we all understand it.
 
Let’s start with the Markets Page Highlights Tab.
 
First off, I imagine we can define a page level query to filter content down to the topic of that page.  For a EU HY Markets page:
 
select * from Content where region[] includes “Europe” and tags[] include “High Yield”
 
All (most) panels on the page inherit that query and then further refine it.  I’ll illustrate that below by querying the page-level resultset.
 
Panel
Query
Most Read Today
title from PageLevelQueryResults order by topRead24 desc limit 10
 
Recent Headlines
title from PageLevelQueryResults where tags[] include “Headline” order by publicationDate desc limit 20
 
Market Movers
title, articleType, markets, sectors, publicationDate from PageLevelQueryResults where tags include “Market Mover” order by publicationDate desc limit 10
 
Trade Ideas
title, articleType, markets, sectors, publicationDate from PageLevelQueryResults where articleType = “Trade Idea” order by publicationDate desc limit 6
 
New Issues
title, articleType, markets, sectors, publicationDate from PageLevelQueryResults where tags include “New Issue” order by publicationDate desc limit 6
 
Market Trends
body from PageLevelQueryResults where tags[] include “Marketing Content” order by publicationDate desc limit 1
 
HY Notes
title from PageLevelQueryResults where articleType = “HY Note” order by publicationDate desc limit 10
 
HY Notes
title from PageLevelQueryResults where articleType = “Company Spotlight” order by publicationDate desc limit 10
 
 
As you move on to Markets Page – CS Picks you can probably write most of these panel queries using small variations on the above.  The Top 10 View Changes panel, though, brings us to a bit  of a different animal: querying for Companies, which I’ve come to understand exist in non-CMS database tables and may deserve their own API.   Or not, as we’d be using the same page-level queries against them as we use for content.  Perhaps the panel would be something like:
 
Select name, rec, recChangeDirection from Company where region[] includes ‘Europe’ and tags[] include ‘High Yield’ order by lastReviewedDate desc limit 10
 
The 8 Results panel on the Companies tab would use a query similar to the above, except the where clause would be formulated by the filter sidebar.  When a use selects one of the listed companies the panels to the right populate with content queries based on the selected company identifier.
 
                Select name, businessDescription from content where articleType = ‘Tearsheet’ and companies[] includes <companyIdentifier> order by publicationDate desc limit 1
 
Recent Coverage:
 
                Select title, articleType from content where companies[] includes <companyIdentifier> order by publicationDate desc limit 10
 
On the Home Page – My CS View tab we see mostly panels like we’ve seen before.  The big diff is that the pageLevelQuery is one formulated from the user’s personalization settings.
 
The Companies tabs present some new wrinkles, not the least of which is what API would one use to discover Related Companies?
 
-Joe

Guys, I extracted a representative sample of page designs from the sprawl for your consideration during the API design process.  The design of these pages will change, though I don’t expect in ways that will totally invalidate any tech or information design work you do early.
 
I’ve included:
-          Markets Page (multiple tabs)
-          Home Page (multiple tabs)
-          Article
-          Company Page (multiple tabs)
 
You’ll see a number of recurring themes here:
-          Pages are built up from panels
-          Panels display list of select content, or directly display a specific content item
-          All (er, most) panels on a page share the same overarching filter criteria, to which they add their own panel-specific refinements
 
There will be Sector pages, but I’m expecting they’ll be built from similar components as the Markets page.
 
-Joe
